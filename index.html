Configurando Git por Primera vez


    git --version
    git config --global user.name "Ing. Leonel Villa"
    git config --global user.username "l2v2blue"
    git config --global user.password "l2v2..2021"
    git config --global user.email l2v2blue@gmail.com
    git config --global user.ui true
    git config --global init.defaultBranch main
    git config --list
    # asignar visual studio code como editor de configuración de git
    git config --global core.editor "code --wait"
    # Abre editor con configuracion generada y espara a que el editor cierre el archivo
    git config --global -e
    # para estandarizar los saltos de línea en windows
    git config --global core.autocrlf true
    # para estandarizar los saltos de línea en linux/mac
    git config --global core.autocrlf input
    # ver todas las opciones de la configuración en la terminal git <comand> -h
    git config -h
    # ver todas las opciones de la configuración en el navegador git help <comand>
    git help config

Flujo Básico

Flujo de Git, consta de tres estados locales y uno de forma remota cuando accedemos al codigo centralizado: modified, staged, committed y remote a cada uno le corresponde un área de trabajo.

  1. Working Directory: Area correspondiente al estado modified, es la carpeta local de PC donde se almacenan los archivos del proyecto.
  2. Staging Area: Es el área correspondiente al estado staged también se le llama index por que es el área donde git indexa y agrega los cambios realizados en los archivos previos a comprometerlos en su registro.
  3. Local Repository: Es el área correspondiente al estado committed, donde los cambios ya se han registrado en el repositorio de git también se le llama HEAD por que indica en qué cambio se encuentra el puntero del repositorio.
  4. Remote Repository: Es el área correspondiente al estado remote y es el directorio remoto donde almacenamos los archivos del proyecto en alguna plataforma web como GitHub, GitLab, BitBucket. Git denomina origin al repositorio remoto.

Master a Main

Repositorios nuevos


  git init
  git add .
  git commit -m "Primer commit"
  git branch -M main
  git remote add origin https://github.com/usuario/repositorio.git
  git push -u origin main

Repositorios existentes


git branch -M main
git remote add origin https://github.com/usuario/repositorio.git
git push -u origin main

Reemplazar la rama master por main en GitHub


# Paso 1
# Crea la rama local main y pásale el historial de la rama master
git branch -m master main

# Paso 2
# Haz un push de la nueva rama local main en el repositorio remoto de GitHub
git push -u origin main

# Paso 3
# Cambia el HEAD actual a la rama main
git symbolic-ref refs/remotes/origin/HEAD refs/remotes/origin/main

# Paso 4
Cambiar rama default de master a main en repositorio de GitHub.

# Paso 5
# Elimina la rama master del repositorio remoto
git push origin --delete master
